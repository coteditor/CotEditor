Set lexical rules in CotEditor on Mac

Lexical rules control how CotEditor interprets delimiter characters while parsing delimiter-based highlight ranges such as strings and block comments.

To adjust lexical rules for a syntax, select Lexical Rules from the list on the left of the syntax editor, then choose a delimiter escape style on the right. See Define a syntax for how to open the syntax editor.

Escape delimiters

Choose how CotEditor escapes delimiters in the document. This setting is used for regular-expression-based syntax highlighting and for matching paired symbols in the editor.

OptionDescription
Delimiter escape style

The style applies to begin/end pairs.

  • Backslash: Treat a backslash (\) as an escape character. A delimiter preceded by a backslash is ignored.
  • None: Do not treat any character as an escape. Every delimiter is matched.

For example, if the escape style is Backslash and the string delimiter is ", a sequence like \" does not end the string.
