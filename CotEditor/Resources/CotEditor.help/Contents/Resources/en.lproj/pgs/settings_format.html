Change Format settings in CotEditor on Mac

Use Format settings in CotEditor to set the default options for documents, such as line endings and text encoding, and to manage syntaxes.

To change these settings in the CotEditor app on your Mac, choose CotEditor > Settings, then click Format.

OptionDescription
Default line endings Select the line ending style used when creating a new document. This setting is also applied when an opened document contains no line endings. For details, see Handle various types of line endings.
Default text encoding Select the text encoding used when creating a new document.
Encoding priorities Set the priority order for text encodings and how CotEditor detects encodings when opening a file.
Edit List
Click Edit List to set the order of encodings shown in the Text Encoding submenu in the Format menu and in the status bar. This order is also used for encoding detection. CotEditor tries each encoding starting from the top of the list. For details, see Change priorities of text encodings.
Refer to encoding declaration
in document		 When this option is enabled, CotEditor takes the encoding declaration written in the document into account when detecting the text encoding. For details, see Use encoding declarations for encoding detection.
Default syntax Select the syntax used when creating a new document.
Available syntaxes The list of all available syntaxes. A dot before the syntax name indicates that the syntax has been customized.
See Define a syntax for editing syntax definitions.
Add +, Delete -
Create a new syntax or delete the selected syntax. Built-in syntaxes can’t be removed.
Edit (pencil icon)
Edit the selected syntax.
Action pop-up menu
  • Duplicate “[syntax name]”: Duplicate the selected syntax to create your own version.
  • Export “[syntax name]”: Export a syntax definition file created from the selected syntax. You can use this exported file to use it on another computer. To use an exported file, choose Import.
  • Share: Share the syntax definition file created from the selected syntax.
  • Import: Import a syntax created on another computer. After choosing Import, select the syntax file you want to add.
  • Show File Mapping Conflicts: Check whether multiple syntaxes are associated with the same extension or filename. This command is unavailable when no conflicts exist.
