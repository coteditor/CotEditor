Write insertion format for File Drop in CotEditor on Mac

CotEditor has the following variables for the use of File Drop.

VariableReplaced with
<<<ABSOLUTE-PATH>>>

The absolute path of the dropped file.

<<<RELATIVE-PATH>>>

The relative path from your document to the dropped file.

<<<FILENAME>>>

The name of the dropped file with extension.

<<<FILENAME-NOSUFFIX>>>

The name of the dropped file without extension.

<<<FILEEXTENSION>>>

The file extension of the dropped file.

<<<FILEEXTENSION-LOWER>>>The file extension of the dropped file (lowercase).
<<<FILEEXTENSION-UPPER>>>

The file extension of the dropped file (uppercase).

<<<DIRECTORY>>>

The directory of the dropped file.

<<<FILECONTENT>>>

The text content of the dropped file (works only when you drop an text file).

<<<IMAGEWIDTH>>>

The image width (works only when you drop an image file).

<<<IMAGEHEIGHT>>>

The image height (works only when you drop an image file).

Use snippets

Here are some examples of what it’s like to use variables.

When you drop a file named “example.html”:

Insertion format you setText actually inserted
<a href="<<<RELATIVE-PATH>>>"></a> <a href="example.html"></a>

When you drop a file named “example.jpg”:

Insertion format you setText actually inserted
<img src="<<<FILENAME>>>" width="<<<IMAGEWIDTH>>>" height="<<<IMAGEHEIGHT>>>" /> <img src="example.jpg" width="160" height="80" />
