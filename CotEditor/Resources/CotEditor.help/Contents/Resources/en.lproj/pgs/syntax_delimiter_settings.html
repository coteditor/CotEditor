Define delimiters in CotEditor on Mac

In CotEditor, you can configure how comment and other delimiters are handled for each syntax. These settings affect not only comment commands but also regular-expression-based syntax highlighting and paired-symbol matching in the editor.

To define delimiters for a syntax, select Delimiters in the list on the left side of the syntax editor, then configure options on the right. See Define a syntax for how to open the syntax editor.

Define comment delimiters

The following comment settings are used for the Comment Out and Uncomment commands. In regular-expression-based syntax definitions, they are also used for comment highlighting.

FieldDescription
Inline comment

Text from the specified delimiter to the end of the line is treated as an inline comment.

Block comment

Text enclosed by the specified begin and end delimiters is treated as a block comment. You need to enter both Begin String and End String.

In the standard Comment Out command, inline comment delimiters are preferred when both inline and block comment delimiters are defined. If multiple delimiters are defined, the first one is used.

Escape delimiters

Choose how CotEditor escapes delimiters in a document. This setting is used for regular-expression-based syntax highlighting and for matching paired symbols in the editor.

OptionDescription
Delimiter escape style

The style applied to begin/end delimiter pairs.

  • Backslash: Treat a backslash (\) as an escape character. A delimiter immediately preceded by a backslash is ignored.
  • None: Do not treat any character as an escape. Every delimiter is matched.

For example, if the escape style is Backslash and the string delimiter is ", a sequence like \" does not end the string.
