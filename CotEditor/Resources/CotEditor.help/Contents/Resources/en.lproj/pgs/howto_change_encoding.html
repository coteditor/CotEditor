Change a document’s text encoding in CotEditor on Mac

CotEditor lets you change the text encoding used when saving a document.

Change the default text encoding

To change the text encoding used for new documents you create in CotEditor, or when opening an empty document, change the “Default text encoding” option in the Format settings.

Change a document’s text encoding

To change a document’s text encoding in CotEditor, do one of the following:

  1. Click the Encoding button in the status bar, or choose Format > Text Encoding.
  2. From the list, select the encoding you want to use.

If the document has never been saved, the encoding you selected is immediately applied.

When you change the encoding of a saved document, a dialog appears.

The options in the dialog are as follows:

OptionDescription
Convert Keeps the current text and changes the document so that it’s saved using the selected encoding.
Reinterpret Reopens the document using the specified encoding. For more information, see “Open a document with a specified encoding.”
Cancel Cancels the operation. The document is not changed.

To change the document’s text encoding, click “Convert.”

If the document contains incompatible characters

If the document contains characters that can’t be represented in the text encoding you’re changing to, a warning dialog appears.

To continue with the change, click “Change Encoding.” The Warnings inspector opens, and you can see a list of characters that are incompatible with the new encoding. For more information, see “Find characters incompatible with the document encoding.”

If you don’t see the encoding you want in the Text Encoding menu

If you don’t see the encoding you want in the Text Encoding menu, you may be able to customize the list and add the encoding you need.

The encodings that appear in the Text Encoding menu can be changed in the Format settings under “Encoding priorities.” For more information, see “Change the priority of text encodings.”
